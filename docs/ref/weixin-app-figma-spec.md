# 贴膜门店小程序 — Figma 设计规范与画板说明

基于 `weixin-app.md` 与 `requirements.md` 整理，支持**用户端（C 端）**与**员工端（门店员工 + 店长）**双视角，便于在 Figma 中逐屏搭建或交给设计师实现。

---

## 端与角色说明

| 端 | 使用人 | 入口 | 核心能力 |
|----|--------|------|----------|
| **用户端** | 车主用户 | 同一小程序，用户身份 | 预约服务、查看进度、订单/施工追踪、评价、个人中心 |
| **员工端** | 门店员工（技师/前台/销售） | 同一小程序，员工身份登录 | 今日任务、工单执行、到店签到、施工阶段更新、拍照、物料扫码、个人绩效 |
| **员工端-店长** | 门店店长 | 同上，店长权限 | 在员工端基础上增加：今日预约总览、预约列表管理、员工/排班快捷入口、当日数据概览 |

---

## 画布与设备

- **画板尺寸**：390 × 844 px（iPhone 14 逻辑分辨率，微信小程序常用）
- **安全区**：顶部留 44px 状态栏，底部留 34px 安全区，内容区约 390 × 766 px
- **背景**：主背景 `#F5F5F5`，卡片/内容区 `#FFFFFF`

## 一、用户端（C 端）画板

### 1.1 首页（Dashboard）

**画板名称**：`C-01-首页`

| 区域 | 高度(约) | 内容 |
|------|----------|------|
| 状态栏 | 44 | 占位或简单色块 |
| 顶部 Banner | 160 | 轮播区：促销 / 保养提醒 / 召回通知（可放 3 个占位矩形） |
| 快捷入口 | 120 | 4 个图标+文案：预约保养、预约维修、预约试驾、紧急救援；网格 2×2，图标 48px |
| 车辆卡片 | 140 | 若已绑定：车型、车牌、下次保养里程/时间、入口「服务历史」；卡片圆角 12，阴影轻 |
| 门店推荐 | 180 | 标题「附近门店」+ 列表 1～2 条：店名、地址、距离、电话；可配小地图占位 |
| 底部导航 | 60 | 首页、预约、我的（仅示意 3 个 tab） |

**主色**：用品牌色宝马蓝（`#0066B1`）做 Banner 点缀、按钮、选中态。

**贴膜场景可替换**：快捷入口可改为「预约贴膜」「服务方案」「优惠券」「我的质保」等，与 requirements 中 C 端能力一致。

---

### 1.2 服务选择页

**画板名称**：`C-02-服务选择`

| 区域 | 说明 |
|------|------|
| 顶部导航 | 返回 + 标题「选择服务」 |
| 分类列表 | 常规保养、维修诊断、钣金喷漆、轮胎/电瓶、其他；每行左侧图标 40px + 标题 + 右箭头 |
| 智能推荐区 | 标题「为您推荐」+ 1～2 个推荐项（含名称、预估价格、所需时间） |
| 服务详情区 | 标题「服务说明」+ 正文：工时、配件、预估价格、所需时间（多行文本占位） |
| 底部 CTA | 主按钮「下一步：选择门店与时间」 |

**贴膜场景**：分类可为全车贴膜、局部贴膜、隐形车衣、改色膜、窗膜等；智能推荐按车型/膜类型匹配。

---

### 1.3 预约填写页

**画板名称**：`C-03-预约填写`

| 区块 | 内容 |
|------|------|
| 选择门店 | 标题 + 地图占位（或列表）：店名、地址、电话、评分、排队情况 |
| 选择时间 | 日历控件占位（7 天） + 时间段（上午 9–12、下午 2–5）+ 剩余名额文案 |
| 车辆信息 | 车牌、VIN（可选）、车型、里程数；标注「自动填充或手动输入」 |
| 备注 | 单行或多行输入占位：「特殊需求，如右前灯不亮」 |
| 联系人 | 姓名、手机号（默认微信绑定，可修改） |
| 底部 | 「提交预约」主按钮 |

**贴膜场景**：可增加「选择技师（可选）」「备注（如需洗车后施工）」等字段。

---

### 1.4 预约确认页

**画板名称**：`C-04-预约确认`

| 区块 | 内容 |
|------|------|
| 汇总卡片 | 服务项目、门店、时间、车辆、联系人（每项一行标签+内容） |
| 费用 | 「预估费用」+ 金额 + 小字「最终以店内检测为准」 |
| 提示 | 「预约成功后将收到短信/微信模板消息通知」 |
| 底部 | 「确认预约」主按钮 + 「取消」次要按钮 |

---

### 1.5 我的预约 / 订单中心

**画板名称**：`C-05-我的预约`

| 区块 | 内容 |
|------|------|
| 顶部 Tab | 全部 | 待服务 | 已完成 | 已取消 |
| 列表项 | 每条：服务类型、门店、日期时间、状态标签；操作「查看详情」「取消」「再次预约」 |
| 空状态 | 无预约时：插画占位 + 「暂无预约」+ 「去预约」按钮 |

**与 requirements 对齐**：列表支持「查看详情」「取消」「再次预约」；详情可跳转施工追踪（阶段状态、施工照片/视频、电子工单/质保卡下载）。

---

### 1.6 施工追踪 / 订单详情

**画板名称**：`C-06-施工追踪`

| 区块 | 内容 |
|------|------|
| 订单概要 | 服务项目、门店、预约时间、状态标签（待到店/施工中/已完成） |
| 进度条 | 阶段化状态：已预约 → 已到店 → 清洗 → 裁膜 → 铺膜 → 收边 → 质检 → 已完成 |
| 施工相册 | 各阶段照片/视频缩略图（由员工上传，可加水印说明） |
| 操作区 | 「下载电子工单」「下载质保卡」（带防伪二维码）、「联系门店」 |

---

### 1.7 个人中心

**画板名称**：`C-07-个人中心`

| 区块 | 内容 |
|------|------|
| 头像区 | 微信头像 + 昵称（可占位） |
| 车辆管理 | 入口「我的车辆」+ 可绑定多辆 |
| 车辆管理 | 入口「我的车辆」+ 可绑定多辆，查看历史施工记录、质保信息 |
| 优惠券/营销 | 入口「我的优惠券」；老带新海报、生日礼包（与 requirements 营销互动一致） |
| 设置列表 | 常用联系人/默认信息、消息通知、客服入口（在线/电话） |
| 底部 | 关于我们、用户协议、隐私政策（文字链接） |

**与 requirements 对齐**：售后与评价—订单完成后可「在线报修」（上传问题照片）、「评价技师与门店」；消息通知为微信服务通知（预约确认、到店提醒、完工通知、质保到期等）。

## 二、员工端（门店员工）画板

员工端与用户端共用画布尺寸与主色，通过**底部 Tab 与首页内容**区分：员工登录后进入「今日任务」而非 C 端首页。

### 2.1 今日任务看板

**画板名称**：`E-01-今日任务`

| 区域 | 高度(约) | 内容 |
|------|----------|------|
| 顶部导航 | 48 | 标题「今日任务」+ 日期选择或「今天」 |
| 统计条 | 40 | 今日预约数、已完成数（可选） |
| 任务列表 | 自适应 | 按时间排序：每条卡片展示 车牌、车型、服务项目、预计到店时间、状态（待到店/已签到/施工中/已完成）；左滑或按钮「签到」「接单」 |
| 空状态 | — | 无预约时：「今日暂无预约」+ 简短说明 |
| 底部导航 | 60 | 任务、工单、我的（员工端 3 个 tab） |

---

### 2.2 到店签到 / 接单

**画板名称**：`E-02-到店签到`

| 区块 | 内容 |
|------|------|
| 客户卡片 | 车牌、车型、服务项目、预约时段、联系人（脱敏）、备注 |
| 操作 | 主按钮「客户已到店，确认签到」；签到后状态变为施工中，可进入工单执行 |
| 次要 | 「联系客户」「联系前台/销售」入口 |

---

### 2.3 工单执行（施工阶段）

**画板名称**：`E-03-工单执行`

| 区块 | 内容 |
|------|------|
| 工单概要 | 车牌、服务项目、客户姓名（脱敏）、当前阶段 |
| 阶段进度 | 横向步骤条：清洗 → 裁膜 → 铺膜 → 收边 → 质检；当前高亮，已完成打勾 |
| 当前阶段操作 | 大按钮「完成本阶段」；本阶段「上传施工照片」（多图，自动加水印：时间+车牌+员工名） |
| 异常标记 | 可选「标记异常」：车身旧伤、客户临时加项等，简短备注 |
| 物料登记 | 入口「扫码登记膜卷」：扫膜卷编号，用于溯源与库存扣减 |
| 底部 | 「全部完成，提交质检」主按钮（当阶段为质检时） |

## 三、员工端（店长视角）画板

店长与员工共用同一员工端小程序，登录后根据角色看到**额外 Tab 或首页入口**：今日预约总览、预约管理、员工/数据概览。

### 3.1 店长首页 / 今日总览

**画板名称**：`M-01-店长首页`

| 区域 | 内容 |
|------|------|
| 顶部 | 标题「店长工作台」或「今日总览」+ 日期 |
| 数据卡片 | 当日预约数、已到店数、施工中数、已完成数；可选：当日营业额、客单价 |
| 快捷入口 | 「预约列表」「排班/员工」「营销发放」「门店设置」（跳转或 H5 后台链接说明） |
| 今日预约列表 | 与员工端「今日任务」类似，但店长可看全部并支持「分配技师」「改时间」等操作（若小程序实现）；否则仅只读总览 |

---

### 3.2 预约列表管理（店长）

**画板名称**：`M-02-预约列表`

| 区块 | 内容 |
|------|------|
| Tab | 全部、待服务、已到店、已完成、已取消 |
| 列表 | 每条：客户车牌、服务项目、预约时间、分配技师、状态；操作「查看」「改派技师」「取消预约」 |
| 筛选 | 按日期、按技师、按服务类型（若需要） |

---

### 3.3 员工 / 排班快捷入口

**画板名称**：`M-03-员工入口`（可为列表或占位）

| 内容 | 说明 |
|------|------|
| 员工列表 | 本店员工姓名、岗位、今日接单数；点击可看该员工今日任务（只读或简单分配） |
| 说明 | 完整「员工档案、提成规则、排班」一般在**商家端管理后台**完成，小程序端仅做快捷查看与分配入口，具体以产品为准 |

---

### 3.4 当日数据概览（店长）

**画板名称**：`M-04-数据概览`

| 区块 | 内容 |
|------|------|
| 核心指标 | 当日营业额、单量、客单价（与 requirements 数据报表一致，店长只看本店） |
| 简表 | 可选：员工绩效排行（今日完成单数）、热门服务 TOP（今日） |
| 说明 | 详细报表在商家端后台；此处为「一眼看懂今天经营情况」的轻量看板 |

## 四、组件与样式建议（用户端 + 员工端共用）

- **主按钮**：高 48px，圆角 24px，品牌主色背景，白字
- **次要按钮**：描边主色，透明底
- **卡片**：背景 #FFF，圆角 12px，阴影 0 2 8 rgba(0,0,0,0.06)
- **标题**：18px 粗体，正文 14px 常规，辅助 12px 灰色
- **图标**：统一 24/48 两档，线条风格

---

## 五、在 Figma 中的操作顺序建议

1. **分组与命名**
   - 建三个顶层分组：**用户端（C）**、**员工端（E）**、**店长端（M）**。
   - 用户端：`C-01-首页` ～ `C-07-个人中心`（共 7 个画板）。
   - 员工端：`E-01-今日任务` ～ `E-07-知识赋能`（共 7 个画板）。
   - 店长端：`M-01-店长首页` ～ `M-04-数据概览`（共 4 个画板）。

2. **画板尺寸**  
   所有画板统一 390 × 844 px，安全区、主色、卡片样式双端一致。

3. **复用与一致性**  
   - 导航栏、底部 Tab、主/次按钮、卡片圆角与阴影做成**组件与样式**，用户端与员工端共用。
   - 员工端与店长端可共用「今日任务/预约列表」等列表卡片，仅通过 Tab 或权限展示不同操作按钮。

4. **主色、圆角、阴影**  
   做成 Color / Effect 样式，便于统一替换品牌色（宝马蓝 `#0066B1`）。

若 Figma MCP 恢复连接，可将上述结构用 `create_frame`、`create_text`、`create_rectangle` 在文档内自动生成；若需某一块拆成更细的坐标与文案，可指定页面名。
